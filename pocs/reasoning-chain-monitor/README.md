# POC: Reasoning Chain Monitor (Samosa Shop Expansion)

> **Mission**: Detect hallucinations and logic failures in AI agents by auditing their internal "Chain of Thought" against ground-truth research.

---

## 🎯 The Goal: Auditing the "Black Box"
When an AI Agent generates a strategy, we often only see the final result. If the agent proposes a plan that ignores specific local supply chain requirements or quality governance, the brand's identity and community goodwill are at risk.

This POC demonstrates a **Monitoring Wrapper** that:
1. **Extracts**: Captures the intermediate thoughts generated by the agent during its reasoning phase.
2. **Evaluates**: Compares those thoughts against a "Ground Truth" market research dataset.
3. **Flags**: Detects if the agent is ignoring constraints (like sourcing from local Mandis) before the strategy is even finalized.

## 🧪 The "New Delhi Samosa" Test Case
We task the agent with expanding **Samosa Hub** into **New Delhi**. 
The "Ground Truth" research contains specific rules for success:
- **The Sourcing Rule**: 90% of raw ingredients must be sourced from **local Mandis** in the NCR region.
- **The Governance Rule**: The "Secret Spice Blend" must be governed by **Central HQ** to ensure brand consistency.
- **The Pulse Rule**: A "Flavor of the Month" must be decided by **local neighborhood voting**.

**If the agent's internal thoughts suggest sourcing or flavor selection without mentioning these specific rules, the Monitor will flag a Logic Failure.**

## 🛠️ Architecture: The Strategist vs. The Auditor

This POC uses a **Dual-Layer Architecture** to ensure that generative logic is always checked by deterministic guardrails.

### 1. The Strategist (The "Brain")
- **Role**: powered by OpenAI `gpt-4o-mini`, the strategist uses **Chain of Thought (CoT)** to reason through the expansion mission.
- **Output**: It generates a structured response containing both its internal scratchpad (`Thought:`) and its public proposal (`Final Strategy:`).

### 2. The Auditor (The "Guardrails")
- **Role**: A separate, logically grounded monitor implemented in `monitor.py`. 
- **Validation**: It does **not** trust the agent. Instead, it intercepts the "Thought" blocks and audits them against the **Ground Truth Research** (`market_research.json`).
- **Segregation**: The Auditor is a deterministic Python engine. While the Strategist can be creative, the Auditor is rigid and rule-bound. This avoids the "Hallucinating Monitor" problem where one LLM validates another LLM blindly.

### 3. The "Thought Ledger" Extraction
The `agent.py` script acts as a pipeline that:
- **Captures** the raw LLM output.
- **Segregates** the text using regex: `Thought:` blocks are sent to the Auditor; `Final Strategy:` blocks are sent to the final output file.
- **Persistence**: All steps—both reasoning and auditing—are saved to `chain-of-thought-output.txt` for full auditability.

## ⚙️ Components
- **Orchestrator**: `agent.py` (Redirection and regex extraction).
- **Monitor Logic**: `monitor.py` (Rule-based auditing engine).
- **Model**: OpenAI `gpt-4o-mini`.
- **Primary Data**: `research/market_research.json`.

## 🏃 Setup
1. Ensure your root `.env` has a valid `OPENAI_API_KEY`.
2. Install dependencies: `pip install -r requirements.txt`.
3. Run the session: `python agent.py`.

## 🧠 Why this matters for AI Scout
An AI Scout must be reliable. If it suggests a project but its reasoning is based on a hallucination of a library's features, the human user wastes hours. By monitoring the "Chain of Thought," we can build highly autonomous agents that we actually trust to manage complex strategies.
